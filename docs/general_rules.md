# 三语提示词构建基本规则 / Үш тілде промпт жасаудың негізгі ережелері / Общие правила создания образовательных промптов

---

## 1. 统一的提示词结构：RCAC 模型 / RCAC моделі: рөл, контекст, әрекет, шектеулер / Единая структура промпта: Модель RCAC

为了确保人工智能在中文、哈萨克语和俄语中输出相同深度和质量的结果，每个提示词都必须遵循以下四部分结构：
ИИ-нің үш тілде де бірдей сапалы және терең жауап беруін қамтамасыз ету үшін әрбір промпт келесі төрт бөлімнен тұруы керек:
Чтобы ИИ выдавал одинаково качественный результат независимо от выбранного языка, каждый промпт должен строиться по следующей четырехуровневой структуре:

| 组件 / Компонент / Компонент | 描述 / Сипаттамасы / Описание | 中文示例 / Қытайша мысал / Пример (ZH) | 哈萨克语示例 / Қазақша мысал / Пример (KK) | 俄语示例 / Орысша мысал / Пример (RU) |
| :--- | :--- | :--- | :--- | :--- |
| **R**ole (角色/Рөл/Роль) | AI 扮演的专业角色 / ИИ кімнің рөлін атқарады / Кем является ИИ | `扮演资深大学教研员...` | `Жоғары оқу орнының тәжірибелі әдіскері ретінде...` | `Действуй как опытный методист вуза...` |
| **C**ontext (上下文/Контекст/Контекст) | 受众、教学目标和背景 / Кім үшін және не үшін жасалады / Для кого и зачем создается материал | `目标受众：一年级本科生，专业是...` | `Мақсатты аудитория: 1-курс студенттері...` | `Целевая аудитория: студенты 1 курса специальности...` |
| **A**ction (行动/Әрекет/Действие) | AI 需要完成的具体任务 / ИИ не істеуі керек / Что конкретно должен сделать ИИ | `针对以下主题设计研讨课教学计划...` | `Тақырып бойынша семинар сабағының жоспарын жаса...` | `Разработай план семинарского занятия на тему...` |
| **C**onstraints (限制/Шектеулер/Рамки) | 输出格式、语气和避免的事项 / Пішім, тон және шектеулер / Ограничения, формат вывода, язык, тон | `采用表格格式。语气：学术、专业。` | `Кесте түрінде пішімде. Тон: академиялық.` | `Форматируй в виде таблицы. Тон: академический.` |

## 2. 语言特性与语气控制 / Тілдік ерекшеліктер және тон / Языковые особенности и тон ИИ

### 🇨🇳 中文 (ZH)
* **常见问题**：AI 可能会输出过于泛化的内容，或使用不符合学术规范的口语化词汇。
* **对策**：在提示词中加入明确的样式约束：*“使用规范的学术中文（教育学专业术语），避免口语化表达。结构清晰，要点明确”*（使用标准的学术中文，避免口语化表达；结构应清晰，重点应突出）。
* **术语规范**：明确指定教学阶段（如“本科一年级”、“高中二年级”），以对齐中国及本地化的课程体系。

### 🇰🇿 Қазақ тілі (KK)
* **Ақаулықтар**: ИИ көбіне ағылшын не орыс тілінен тікелей аударма жасап, стилистикалық қателер жібереді немесе тым қысқа жауап қайтарады.
* **Шешімі**: Промпт мәтініне мынадай нұсқауларды міндетті түрде қосу керек: *«Қазақ әдеби тілі мен академиялық стиль нормаларын қатаң сақта. Аударма сипаттағы сөйлемдерден қаш. Толық және мазмұнды жауап бер»*.
* **Терминология**: Ресми білім беру терминдерін қолдануды талап ету (мысалы, *«білім алушы»*, *«күтілетін нәтиже»*, *«кері байланыс»*).

### 🇷🇺 Русский язык (RU)
* **Проблема**: Склонность ИИ к использованию громоздких канцеляризмов, пассивного залога и пустых вводных слов.
* **Решение**: Добавлять в промпт правила: *«Пиши живым профессиональным языком, избегай шаблонных вводных фраз и канцеляризмов. Используй активный залог»*.
* **Стандарты**: Требовать явного соответствия принятой академической терминологии и целям по таксономии Блума.

## 3. 提示词中的变量配置 / Промпттердегі айнымалыларды баптау / Переменные в промптах

所有需要教师替换的占位符均使用方括号标注，并按如下语序翻译：
Оқытушы өз деректерімен ауыстыруы керек айнымалылар тізбегі келесідей беріледі:
Все переменные, которые преподаватель должен заменить на свои данные, стандартизированы и выделены квадратными скобками:

* **[学科 / Пән / Предмет]**
* **[主题 / Тақырып / Тема]**
* **[阶段 / Деңгей / Уровень]**
* **[时长 / Ұзақтығы / Продолжительность]**
* **[格式 / Формат / Формат]**
* **[区域聚焦 / Аймақ / Регион-Фокус]**

## 4. 输出质量控制规则 / Жауап сапасын бақылау ережелері / Правила контроля качества вывода

1. **深度控制 / Тереңдікті бақылау / Контроль глубины**：
   * 明确指定生成数量。例如：*“设计不少于5个实践任务... / Кемінде 5 практикалық тапсырма дайында... / Сгенерируй не менее 5 практических заданий...”*.
2. **禁止占位符 / Қысқартуларға тыйым салу / Запрет на сокращения**：
   * *“严禁使用诸如‘以此类推’、‘后续步骤略’等占位符，必须完整写出所有步骤... / Жауапты толық және мазмұнды жазып, ‘осы сияқты жалғастыру’ деген сияқты қысқартуларды қолданба... / Не используй заглушки вида ‘и так далее’ или ‘остальной текст по аналогии’. Напиши все разделы полностью”*.
3. **结构化标记 / Құрылымдық белгілеу / Единый разделитель (Markdown)**：
   * 使用 Markdown 标题 `#`, `##` 和列表进行多级结构化排版，便于复制和二次编辑。
